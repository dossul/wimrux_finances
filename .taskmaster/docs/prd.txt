SECTION 1: CONTEXTE ET VISION
================================================================================

## 1.1 Entreprise
- Nom: WESTAGO SARL
- Pays: Burkina Faso
- Réglementation: Direction Générale des Impôts (DGI)
- Obligation légale: Mise en conformité SECeF (Système Électronique Certifié de Facturation) au 1er Juillet 2026

## 1.2 Objectif
Livrer un SFE (Système de Facturation d'Entreprise) homologué par la DGI du Burkina Faso, sous forme d'application web SaaS multi-entreprise.

## 1.3 Règle d'Or (NON-NÉGOCIABLE)
Le logiciel DOIT garantir:
- INALTÉRABILITÉ des données de facturation
- SÉCURISATION (chiffrement AES-256, signatures)
- CONSERVATION et ARCHIVAGE
- INTERDICTION ABSOLUE de supprimer ou modifier une facture validée/certifiée
- INTERDICTION ABSOLUE de modifier un log d'audit

- AI: DeepSeek v3.2, GPT-4o-mini, Claude Sonnet 4.5, Gemini 3 Pro, Grok 4.1, MiniMax M2.1

## 3.6 Règles de Calcul Fiscal

### Mode TTC (prix incluant TVA):
- Montant HT = Prix TTC / (1 + Taux TVA)
- Montant TVA = Prix TTC - Montant HT

### Mode HT (prix hors TVA):
- Montant TVA = Prix HT × Taux TVA
- Montant TTC = Prix HT + Montant TVA

### PSVB (Prélèvement à la Source sur Vente de Biens):
- PSVB = Montant TTC × Taux PSVB du groupe
- Groupe A: 2%, B: 2%, C: 2%, D: 0%, E: 1%, F: 1%, G: 1%, H-P: 0%

### Timbre Quittance (paiements espèces uniquement):
| Montant TTC | Timbre |
|-------------|--------|
| < 5 000 FCFA | 0 FCFA |
| 5 000 – 25 000 | 100 FCFA |
| 25 001 – 50 000 | 200 FCFA |
| 50 001 – 100 000 | 500 FCFA |
| > 100 000 | 1 000 FCFA |

### Arrondis:
- 2 décimales, arrondi supérieur si ≥ 0.005
- En cas d'arrondi fiscal: Montant HT + Montant Taxe = Montant TTC (toujours)

| Code | Description |
|------|-------------|
| BF001 | IFU vendeur absent ou invalide |
| BF002 | Type de facture non valide |
| BF003 | Référence de facture manquante ou invalide |
| BF004 | Articles manquants (minimum 1 requis) |
| BF005 | Groupe de taxation invalide |
| BF006 | Type de client invalide |
| BF007 | IFU client obligatoire pour type PM ou PC |
| BF008 | Mode de paiement invalide |
| BF009 | Montant total paiement ≠ montant TTC |
| BF010 | Montant article ou facture négatif ou nul |
| BF011 | Type d'article invalide |
| BF012 | Limite de factures en attente atteinte |
| BF013 | Facture déjà confirmée ou annulée |
| BF014 | Facture introuvable |
| BF015 | Dispositif SECeF bloqué (connexion requise) |
| BF016 | Dispositif SECeF désactivé |
| BF017 | NIM non enregistré ou invalide |
| BF018 | Référence de facture en double |
| BF019 | Format adresse cadastrale invalide (SSSS LLL PPPP) |
| BF020 | Mode de prix invalide (doit être HT ou TTC) |

## 3.9 Schéma Base de Données pour l'API Simulée

```sql
CREATE TABLE devices (
  nim VARCHAR(10) PRIMARY KEY,
  ifu VARCHAR(20) NOT NULL,
  status VARCHAR(20) NOT NULL DEFAULT 'ACTIF',
  jwt_secret VARCHAR(255) NOT NULL,
  activation_counter INTEGER DEFAULT 1,
  last_audit_remote TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE sim_invoices (
  uid UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nim VARCHAR(10) NOT NULL REFERENCES devices(nim),
  ifu VARCHAR(20) NOT NULL,
  reference VARCHAR(100) NOT NULL UNIQUE,
  type VARCHAR(2) NOT NULL CHECK (type IN ('FV','FT','FA','EV','ET','EA')),
  status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  fiscal_number VARCHAR(50),
  code_secef_dgi VARCHAR(29),
  qr_code TEXT,
  signature TEXT,
  data JSONB NOT NULL,
  tax_calculation JSONB,
  date_submitted TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  date_certified TIMESTAMP,
  counters VARCHAR(50)
);
```

SECTION 4: MENTIONS OBLIGATOIRES SUR LA FACTURE
================================================================================

Selon les spécifications techniques de la DGI (Note n°2025-0889/MEF/SG/DGI):

## 4.1 Informations Vendeur
- Nom de l'entreprise
- IFU de l'entreprise
- Adresse avec références cadastrales (Section, Ilot, Parcelle: SSSS LLL PPPP)
- Contact (téléphone, email)
- Compte(s) bancaire(s)
- Régime d'imposition
- Service des impôts de rattachement

## 4.2 Informations Client
- Type de client (CC, PM, PP, PC)
- IFU et nom (obligatoire pour PM et PC)
- Adresse et contact (optionnel)

## 4.3 Éléments de la Facture
- Numéro de série unique (série ininterrompue par année)
- Date et heure d'établissement
- Type de facture (FV, FT, FA, EV, ET, EA)
- Mention "DUPLICATA" si copie
- Mention "FACTURE D'AVOIR" + nature si avoir
- Mention "EXPORTATION" si export
- Mention "ACOMPTE" si acompte

## 4.4 Détails Articles
- Désignation (min 64 caractères de capacité)
- Type d'article (LOCBIE, LOCSER, IMPBIE, IMPSER)
- Quantité, unité de mesure
- Prix unitaire, montant
- Groupe de taxation
- Réductions éventuelles

## 4.5 Calculs Fiscaux sur la Facture
- Mode de prix (HT ou TTC)
- Montants totaux HT par groupe de taxation
- Taux d'imposition appliqués
- Montants de TVA par groupe
- Montants de PSVB si applicable
- Taxe spécifique si applicable
- Montant total TTC
- "Montant timbre quittance" + montant si paiement espèces
- Montant total en lettres

## 4.6 Paiement
- Mode(s) de paiement
- Montant par mode

## 4.7 Éléments de Sécurité (du MCF/API FNEC)
- Code SECeF/DGI (24 caractères avec tirets: XXXX-XXXX-XXXX-XXXX-XXXX-XXXX)
- NIM (numéro machine, format MMCCNNNNNN, 10 caractères)
- ISF (identifiant système)
- Compteurs de factures
- QR Code 2D (format: BF;fiscalNumber;codeSECeF;IFU;dateTimeCompact)
- Date et heure de certification (celle du MCF, pas du serveur)
- Mention: "EXIGEZ LA FACTURE ÉLECTRONIQUE CERTIFIÉE"

## 4.8 Autres
- Nom de l'opérateur
- Lignes de commentaire (max 8)
- Référence certificat d'exonération si applicable
- Base juridique si exonération

================================================================================
SECTION 7: SÉCURITÉ
================================================================================

## 7.1 RLS (Row Level Security)
- Toutes les tables métier ont RLS activé
- Politique: user ne voit que les données de sa company_id
- audit_log: INSERT only par système, SELECT par admin/auditeur

## 7.2 Triggers d'Inaltérabilité
- Trigger BEFORE UPDATE/DELETE sur `invoices`: REJETER si status IN ('validated', 'certified')
- Trigger BEFORE UPDATE/DELETE sur `audit_log`: REJETER TOUJOURS

## 7.3 Chiffrement
- AES-256 CBC pour IFU, clés API, données MCF
- TLS pour toutes les communications

================================================================================
SECTION 8: PROCESSUS DE FACTURATION COMPLET
================================================================================

1. Caissier crée un brouillon (status: 'draft')
2. Ajoute lignes d'articles, sélectionne client, mode prix
3. Calculs fiscaux en temps réel (TVA, PSVB, timbre)
4. Caissier clique "Valider" → point de non-retour
5. Status passe à 'validated' (trigger bloque UPDATE/DELETE)
6. Driver useFnecApi.submitInvoice() → reçoit UID + calculs serveur
7. Vérification: calculs client = calculs serveur
8. Driver useFnecApi.confirmInvoice(uid) → reçoit éléments de sécurité
9. Stockage: code_secef_dgi, qr_code, signature, nim, counters, fiscal_number
10. Status passe à 'certified'
11. Génération PDF avec bloc sécurité complet
12. Stockage PDF dans InsForge Storage
13. Entrée dans audit_log (action: 'invoice_certified')
14. Si mode dégradé: étapes 6-12 sont différées via pending_certification_queue

================================================================================
SECTION 10: CONTRAINTES ET RÈGLES NON-FONCTIONNELLES
================================================================================

- Langue de l'UI: Français
- Monnaie: FCFA (pas de centimes dans l'affichage, mais 2 décimales en calcul)
- Fuseau horaire: UTC+0 (Burkina Faso GMT)
- Performance: latence max 3 secondes pour certification
- Disponibilité: mode dégradé si API FNEC indisponible
- PWA: l'application doit fonctionner comme PWA sur mobile
- Responsive: mobile-first design
- Tests: chaque module doit avoir des tests unitaires
- TypeScript strict: pas de `any`, interfaces typées pour toutes les données
- Composition API: pas d'Options API Vue 2
- InsForge SDK: toujours utiliser le SDK, jamais d'appels HTTP directs à InsForge
